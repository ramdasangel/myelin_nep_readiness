4. Functional Flow — Three Phases

Phase 1

Task Mapping

User selects 3–12 tasks to practice over 21 days
Phase 2

Daily Logging

User checks in daily to mark practiced tasks + reflect
Phase 3

Progress Tracking

Dashboard shows engagement, completion, and consistency

Phase 1: Task Mapping (Enrollment)

  1. User logs in and navigates to the Micro-Intervention module
  2. System displays all 12 tasks organized by FP — each with name, description, FP tag, depth tag. Bilingual toggle: EN | मराठी
  3. User selects 3–12 tasks. Selection highlights FP coverage and depth distribution visually
  4. User confirms selection → "Start My Practice"
  5. System saves to UserTaskMapping: userTempId, SelectedTasks[], createdAt
  6. User sees confirmation screen with selected tasks and practice start date
Business Rules: Minimum 3 tasks, maximum 12. User can re-select at any time (overwrites previous). Same task library for Teachers and Leaders.

Phase 2: Daily Logging (21-Day Practice Window)

  1. User opens "Daily Practice Log" screen. Current date shown prominently
  2. System displays only the user's selected tasks (not all 12). Each task rendered as a card with checkbox + optional comment field
  3. User checks off completed tasks → true (practiced) / false (not practiced)
  4. User optionally adds a reflection comment (free text)
  5. System auto-saves on each interaction (no submit button)
  6. System creates UserDailyProgress record: UserTempId, SubmitDate, TasksProgress[]
  7. User sees visual confirmation — checkmarks turn green
Business Rules: One log per user per day. All selected tasks appear (checked or unchecked). Comments optional. Past dates cannot be edited. All entries are append-only (no deletion).

Phase 3: Progress Tracking (Dashboard)

  1. User navigates to "My Progress" screen
  2. System displays a 21-day progress grid: Rows = tasks, Columns = dates. Cell states: ✓ Done | ✓+comment | ✗ Not done | — Future
  3. System calculates per-user metrics: completion rate, consistency %, comment rate
  4. Admin dashboard aggregates across all users: KPIs, charts (daily timeline, task completion, FP radar), filterable user table

8. Design Principles

💬

Trust-Based Reflection

Self-reported check-ins, not surveillance. No external validation required.

⚡

Minimal Friction

Auto-save on interaction. No submit button. Card-based mobile UI.

🇧

Bilingual From Day 1

English + Marathi toggle on all screens. Tasks, descriptions, UI labels.

📱

Mobile-First

Cards on mobile, tables on desktop. Responsive layout throughout.

🔒

Append-Only Data

No edits or deletes. All log entries are permanent and immutable.

🤝

Role-Agnostic Tasks

Same 12-task library for Teachers and Leaders. Context differs, tasks don't.
