Adaptive Mechanism

Each of the 11 topics contains exactly 3 questions at difficulty levels 1, 2, and 3 (30 total). The adaptive engine selects 11 questions — not all topics are guaranteed to appear for every respondent. Some topics may get 1 question, others 2–3, depending on the person's performance path. This means not every competency area is assessed for every person.

Question Bank Structure (per language): 30 questions total ├── 11 Topics × 3 Levels each │ ├── Level 1 (Foundational / Easy) — 1 mark │ ├── Level 2 (Preparatory / Medium) — 1 mark │ └── Level 3 (Middle Stage / Hard) — 1 mark │ └── Adaptive Engine selects 11 of 30 └── Weighted scoring: Level 1 = 3 marks, Level 2 = 2 marks, Level 3 = 1 mark Total possible = 32 marks (varies by selection)

userOptionSequence — Answer Change Tracking

Each question response may contain a userOptionSequence array that records every option the respondent clicked, with timestamps:

Example: userOptionSequence [ { "optionText": "3", "time": 2, "sequenceNo": 1 }, ← first click { "optionText": "4", "time": 3, "sequenceNo": 2 }, ← changed to "4" { "optionText": "12", "time": 4, "sequenceNo": 3 }, ← changed to "12" { "optionText": "6", "time": 5, "sequenceNo": 4 }, ← changed to "6" { "optionText": "4", "time": 8, "sequenceNo": 5 } ← back to "4" (flip-flop!) ] Final answer = last entry ("4") Changes = sequenceNo count − 1 = 4 Flip-flop = YES (returned to "4" which was seen at seq 2) Deliberation time = time[last] − time[first] = 8 − 2 = 6 seconds

4. Scoring & Ranking

Raw Score

Per-person score Obtained Marks = Σ (marks for each correctly answered question) Marks per question vary by difficulty level: Level 1 (Easy) → 3 marks Level 2 (Medium) → 2 marks Level 3 (Hard) → 1 mark Total Possible Marks = 32 (sum of marks for the 11 served questions)
Note: The marks weighting is inverse to difficulty — Level 1 (easy) questions carry the most marks (3), while Level 3 (hard) carry the least (1). This means the assessment rewards foundational mastery more heavily than advanced problem-solving.

Percentage Score

Formula Percentage = (Obtained Marks / Total Test Marks) × 100 where Total Test Marks = 32 (pre-computed in the system)

Competency Area Accuracy

Per-person per-area Area Accuracy(%) = (Questions Right in Area / Total Questions Served in Area) × 100 Note: "Questions Served" varies per person due to adaptive selection. A person may get 0, 1, 2, or 3 questions in any given area.

Aggregate Area Accuracy (for groups)

Group-level (e.g., all Teachers) Group Area Accuracy(%) = (Σ Right answers across all people in area) / (Σ Total questions served across all people in area) × 100
Example: If 75 teachers collectively answered 78 Multiplication questions and got 71 right:
Teacher Multiplication Accuracy = 71 / 78 × 100 = 91.0%

Ranking

Respondents are ranked by Percentage score descending. Ties share the same effective percentage but receive adjacent rank numbers.

7. NEP Exit Level Framework

Concept

The National Education Policy (NEP) 2020 defines four stages of school education. The Mathangle assessment content spans Preparatory and Middle stage mathematics. We derive an exit level per competency area — the highest stage at which the respondent demonstrated mastery.

NEP School Education Stages: ┌──────────────────┐ │ Foundational │ Ages 3–8 (Grades Pre-K to 2) │ Stage │ Basic number sense, shapes, patterns ├──────────────────┤ │ Preparatory │ Ages 8–11 (Grades 3 to 5) │ Stage │ Arithmetic operations, measurement, basic geometry ├──────────────────┤ │ Middle │ Ages 11–14 (Grades 6 to 8) │ Stage │ Factors/multiples, advanced geometry, word problems ├──────────────────┤ │ Secondary │ Ages 14–18 (Grades 9 to 12) │ Stage │ Not assessed in Mathangle └──────────────────┘

Mapping Question Levels to NEP Stages

Each topic has 3 questions at levels 1, 2, 3. We map these to NEP exit stages:

Question LevelNEP Exit StageColor CodeInterpretation
Level 1 (Easy) Foundational Amber Can handle basic, straightforward application of the concept. Has exited the foundational stage for this skill.
Level 2 (Medium) Preparatory Blue Can handle multi-step problems. Has exited the preparatory stage for this skill.
Level 3 (Hard) Middle Green Can handle advanced, non-routine problems. Has exited the middle stage for this skill.
All wrong (served but failed) Below Foundational Red Could not demonstrate mastery at any level for this competency.
Not served by adaptive engine Not Assessed Gray The adaptive engine did not select questions from this competency area.

Exit Level Derivation Algorithm

Algorithm: Per-person per-competency exit level For each person P and each competency area A: 1. Collect all questions served to P in area A (via Subtopic → Competency mapping from the question bank) 2. For each question, record: - Question_Level (1, 2, or 3) - Whether it was answered correctly (ObtainedMarks > 0) 3. If multiple subtopics map to the same competency (e.g., Geometry has "Types of angles and shapes" AND "Measuring angles"), take the BEST result at each level across subtopics. 4. Find the HIGHEST level where at least one question was answered correctly: - If Level 3 correct → Exit Stage = "Middle" - Else if Level 2 correct → Exit Stage = "Preparatory" - Else if Level 1 correct → Exit Stage = "Foundational" - Else if questions were served but all wrong → "Below Foundational" - Else (no questions served) → "Not Assessed"
Example: Person X gets these questions in Geometry:
• Level 2 (Measuring Angles): ✓ Correct
• Level 3 (2D/3D Shapes): ✗ Wrong

Highest correct level = 2 → Exit Stage = Preparatory

Even though they attempted Level 3, they only demonstrated mastery at Level 2.

8. Indecision Index

Purpose

The Indecision Index quantifies how much a respondent wavered between answer choices during the assessment. It captures uncertainty, second-guessing, and flip-flopping behavior using the userOptionSequence data recorded per question.

Raw Metrics (per question)

MetricDefinitionExample
Number of Changes len(userOptionSequence) − 1
0 if sequence length ≤ 1		 Sequence [A, B, C] → 2 changes
Flip-Flop Boolean: did the respondent return to a previously selected option? Sequence [A, B, A] → flip-flop = true
Sequence [A, B, C] → flip-flop = false
Deliberation Time time[last entry] − time[first entry]
Seconds spent actively changing answers		 Sequence times [2, 3, 8] → 6 seconds
Changed From Correct First choice was the correct answer, but final choice was wrong Correct = "B", Sequence [B, C] → changed from correct = true
Changed To Correct First choice was wrong, but final choice was the correct answer Correct = "C", Sequence [A, C] → changed to correct = true

Aggregate Metrics (per person)

Aggregation across all 11 questions Total Changes = Σ (changes per question) Total Flip-Flops = count of questions where flip-flop = true Changes Per Question = Total Changes / Total Questions Served Total Time = Σ (TimeTaken per question) Change Time = Σ (Deliberation Time per question with changes) Time Ratio = min(Change Time / Total Time, 1.0) Changed From Correct = count of questions where first choice was correct but final was wrong Changed To Correct = count of questions where first choice was wrong but final was correct

Indecision Score Formula

Indecision Index (0–100 scale) ┌─────────────────────────────────────────────────────────────┐ │ │ Indecision Score = │ min(Changes_Per_Q / 4, 1.0) × 40 ← Raw Changes │ │ (40% weight) │ │+ Flip_Flop_Rate × 25 ← Flip-Flops │ │ (25% weight) │ │+ Time_Ratio × 15 ← Deliberation │ │ (15% weight) │ │+ Changed_From_Correct_Rate × 20 ← Harmful Change │ │ (20% weight) │ │ │ └─────────────────────────────────────────────────────────────┘ where: Changes_Per_Q = Total Changes / Total Questions Flip_Flop_Rate = Total Flip-Flops / Total Questions Time_Ratio = min(Change Time / Total Time, 1.0) Changed_From_Correct_Rate = Changed From Correct / Total Questions Final Score = min(Score, 100) (capped at 100)

Component Weights — Rationale

ComponentWeightWhy
Raw Changes 40% The most direct measure of indecision. Normalized by dividing by 4 (maximum practical changes per question, given 4 MCQ options). More changes = more uncertainty.
Flip-Flops 25% Returning to a previously rejected option is a stronger signal of indecision than linear exploration. It indicates genuine confusion rather than methodical elimination.
Deliberation Time 15% Time spent actively changing answers (as a ratio of total time). Low weight because some slow changers are being thoughtful, not indecisive.
Changed From Correct 20% The most harmful type of indecision — the respondent initially knew the answer but second-guessed themselves into a wrong answer. This represents self-doubt undermining knowledge.

Interpretation & Color Coding

Score RangeColorLabelInterpretation
0 ■ Green Decisive Never changed any answer. Committed to first choice on all questions.
0.1 – 5.0 ■ Yellow-Green Mild Occasional answer changes (1–2 questions). Normal and often beneficial — reconsideration led to corrections.
5.1 – 10.0 ■ Amber Moderate Multiple answer changes, possibly with flip-flops. Suggests uncertainty in several topic areas.
> 10.0 ■ Red High Frequent changes, flip-flops, and/or harmful changes (correct→wrong). Significant uncertainty or test anxiety.
Example Calculation:

Person answered 11 questions. On 2 questions they changed answers:
  • Q5: Changed once (seq length 2), no flip-flop, deliberation = 19 sec, was wrong→correct ✓
  • Q8: Changed 3 times (seq length 4), flip-flop = yes, deliberation = 51 sec, was correct→wrong ✗

Aggregates:
  • Total Changes = 1 + 3 = 4
  • Changes Per Q = 4 / 11 = 0.364
  • Flip-Flop Rate = 1 / 11 = 0.091
  • Time Ratio = 70 / 480 = 0.146 (total test time was 480 sec)
  • Changed From Correct Rate = 1 / 11 = 0.091

Score:
  • min(0.364/4, 1.0) × 40 = 0.091 × 40 = 3.64
  • 0.091 × 25 = 2.27
  • 0.146 × 15 = 2.19
  • 0.091 × 20 = 1.82
Total Indecision Score = 3.64 + 2.27 + 2.19 + 1.82 = 9.9 → Moderate (Amber)

Net Harm Metric

Supplementary metric Net Harm = Changed From Correct − Changed To Correct Positive → answer changes hurt the person overall (more marks lost than gained) Negative → answer changes helped (more corrections than regressions) Zero → neutral effect

10. Limitations & Caveats

  1. Adaptive Selection Bias: Not all 8 competency areas are assessed for every person. Some persons may have "Not Assessed" for 2–3 areas. Comparisons across persons for a specific area are valid only among those who received questions in that area.
  2. Small Question Count per Area: Most areas have only 1–3 questions served per person. A single right/wrong answer can swing the area accuracy by 33–100%. Individual area scores should be interpreted cautiously; aggregate (branch/group) scores are more reliable.
  3. Inverse Mark Weighting: Easy questions (Level 1) carry 3 marks while hard questions (Level 3) carry 1 mark. This means the percentage score rewards foundational mastery disproportionately. Two persons with the same number of correct answers can have very different percentages depending on which levels they got right.
  4. Exit Level = Highest Demonstrated, Not Guaranteed: A person who got Level 3 correct might have gotten Level 1 wrong (due to adaptive selection skipping it). The exit level reflects the highest demonstrated capability, not a comprehensive mastery assessment at every level below.
  5. Indecision ≠ Incompetence: A high indecision score does not necessarily indicate low competence. Some respondents change answers to arrive at correct ones (beneficial indecision). The "Changed To Correct" vs "Changed From Correct" breakdown distinguishes helpful reconsideration from harmful second-guessing.
  6. 13 Missing Sequences: 13 out of 1217 answer records did not have userOptionSequence data (likely due to technical issues or instant submission). These are treated as having 0 changes.
  7. Record Count Difference: 109 records in the analysis collection (OnlineAssesmentAnalysisStudentLevel) vs 121 in the raw results (OnlineAssesmentStudentResult). The difference of 12 may be duplicate submissions or re-attempts. The dashboard uses analysis data for scoring/ranking and raw results for exit levels and indecision.
  8. TimeTaken Anomalies: Some TimeTaken values are negative (minimum observed: −98 seconds), likely due to clock synchronization issues. These are used as-is in deliberation time calculations but have minimal impact on the indecision index due to the 15% weight cap.
